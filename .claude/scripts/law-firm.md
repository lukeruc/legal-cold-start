# 律所律师冷启动脚本

本文件在 CLAUDE.md 检测到未配置状态、用户选择"律所律师"后加载。包含对话大纲、种子文件提取指南，完成后写入档案（使用 `.claude/profiles/law-firm.md` 模板）。

---

## 冷启动对话

**你的用户是律所律师。** 需要了解：执业领域、团队结构、客户类型、事项管理方式。不需要问"立场"（随委托变化，每次审查时确认）。

### 种子文件（优先）

在开始任何问题之前，先问：

> 在开始之前——如果你手头有常用文书范本（起诉状、法律意见书、尽调报告、合同模板），分享给我。我可以从中了解你的业务领域、法域和写作风格，后面只问文件和你说不清楚的事。没有也没关系，我们聊几分钟就行。

如果用户给了种子文件，提取：
- 文书类型分布 → 业务领域（诉讼/非诉/顾问，及细分方向）
- 法院/仲裁机构 → 管辖法域 + 纠纷处理层级
- 引用法规体系 → 法域指标
- 语言和写作风格 → 文书风格偏好
- 文件中反复出现的当事人特征 → 客户类型线索（大企业、中小、个人）

确认后，只问文件回答不了的：团队规模与分工、客户类型与管理方式、输出偏好、常用工具、事项管理习惯。

### 纯对话模式（无种子文件时）

每轮 ≤ 3 个问题。

**Part 0：执业画像**

> 在什么律所？什么职位（合伙人/律师/实习律师）？

> 主要做哪些业务？诉讼（什么方向）、非诉（并购/IPO/尽调）、常年顾问？

> 主要在哪些法域和法院/仲裁机构执业？

**Part 1：客户**

> 你的客户主要是什么类型？大国企、上市公司、中小企业、个人？

**Part 2：工具验证**

> 常用哪些工具？法律数据库？工商查询？文档处理？和法院系统对接的电子送达/立案平台？

实际调用测试每个工具，报告 ✓ ⚪ ✗。方法同 in-house 脚本。

**Part 3：输出偏好**

> 不同类型的文书偏好什么风格？诉讼文件（起诉状、代理词）和顾问意见（法律意见书、尽调报告）的风格可能不同。

> 风险/结论评级体系？引用格式偏好？

对话完成后：总结确认 → 写入档案（**不保留 STATUS: UNCONFIGURED**）→ 完成后的 handoff。

写入后清理：
- 删除 `.claude/scripts/` 整个目录
- 删除 `.claude/profiles/` 整个目录

### 暂停支持

用户说 "pause" → 已回答的写入档案，未答的标 `[PENDING]`，文件顶部加 `<!-- SETUP PAUSED AT: <section> -->`。

## 种子文件提取指南（律所律师版）

### 业务领域识别

- 诉讼类：起诉状、答辩状、代理词、上诉状 → 进一步按案由（民商事/刑事/行政）
- 非诉类：尽调报告、交易文件、法律意见书 → 进一步按交易类型
- 顾问类：合同审核意见、日常咨询回复、合规意见

### 法域与管辖识别

搜索法院名称（"XX人民法院""XX District Court"）、仲裁机构（"贸仲""CIETAC""HKIAC"）、适用法律条款。

### 客户类型线索

文书中当事人身份：上市公司/自然人以个人名义/中小企业/政府机关。

### 来源记录

```
*提取自种子文件：15 份文件中 8 份为民商事诉讼（起诉状 3 份、代理词 3 份、上诉状 2 份），4 份为并购尽调意见，3 份为常年顾问合同审核意见。客户以制造业和房地产企业为主。用户确认为民商事诉讼和并购非诉方向。*
```
