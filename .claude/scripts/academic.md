# 学术研究者冷启动脚本

本文件在 CLAUDE.md 检测到未配置状态、用户选择"学术研究者"后加载。包含对话大纲、来源文件提取指南，完成后写入档案（使用 `.claude/profiles/academic.md` 模板）。

---

## 冷启动对话

**你的用户是学术法律研究者。** 需要了解：研究领域、机构、方法论、引用体系、产出类型。学术研究者没有客户，不代理案件——工作对象是论文、课题、教学内容。

### 来源文件（优先）

在开始任何问题之前，先问：

> 在开始之前——如果你手头有近期写的论文、讲义、研究计划书、课题申报材料，分享给我。我可以从中了解你的研究领域、法域和引用风格，后面只问文件和你说不清楚的事。没有也没关系。

如果用户给了来源文件，提取：
- 论文/讲义的主题分布 → 研究领域
- 引用格式（MLA/Bluebook/Chicago/各校自主格式）→ 引用风格
- 法规体系引用模式 → 主要研究的法域
- 语言和学术写作风格 → 产出风格偏好
- 文件中反复出现的课题方向 → 持续研究主题

确认后，只问文件回答不了的：研究机构与角色、团队协作方式、常用工具、正在进行的项目。

### 纯对话模式（无来源文件时）

每轮 ≤ 3 个问题。

**Part 0：研究画像**

> 在什么机构？大学法学院/研究机构/独立学者？什么角色（教授/研究员/博士生/博士后）？

> 主要研究哪些领域？中国法/比较法/国际法？哪个方向（宪法/民法/刑法/法理学/知产/经济法……）？

**Part 1：写作与发表**

> 主要研究哪些法域？中国大陆/普通法系/欧盟法/比较法？

> 主要产出什么类型的文字？论文（中文核心期刊/SSCI/会议论文）、教材/专著、政策研究报告？

> 目标发表载体是哪些？法学研究/中国法学？还是 SSCI 法学期刊？有什么引用格式要求？

**Part 2：工具验证**

> 常用哪些工具？法律数据库？文献管理？数据分析？

实际调用测试每个工具，报告 ✓ ⚪ ✗。方法同 in-house 脚本。

对话完成后：总结确认 → 写入档案（**不保留 STATUS: UNCONFIGURED**）→ 完成后的 handoff。

写入后清理：
- 删除 `.claude/scripts/` 整个目录
- 删除 `.claude/profiles/` 整个目录

### 暂停支持

用户说 "pause" → 已回答的写入档案，未答的标 `[PENDING]`，文件顶部加 `<!-- SETUP PAUSED AT: <section> -->`。

## 来源文件提取指南（学术版）

### 研究领域识别

从论文标题、摘要、引言和引用文献中判定：
- 主题关键词聚类 → 研究领域（宪法/民法/刑法/商法/知识产权/法理学/国际法/经济法…）
- 引用法规体系 → 研究的法域（中国法/美国法/EU 法/国际法）
- 引用学说体系 → 学术流派/理论框架

### 引用风格识别

提取文件中法律引用的格式，判定常用引用体系：
- 条文号格式（"《民法典》第X条"/"28 U.S.C. § X"/"Art. X GDPR"）
- 案例引用格式（"(2025)最高法民终X号"/"X F.3d X (2d Cir. 2024)"）
- 学术引用格式（脚注 vs 文内引用 vs 参考文献列表）

### 来源记录

```
*提取自来源文件：3 篇论文中 2 篇为公司法方向（股东权利/公司治理），1 篇为证券法方向。引用格式为中文法学引注手册（脚注），法条引用兼含中国大陆和比较法（德国GmbHG、美国DGCL）。用户确认为公司法/证券法比较研究方向。*
```
