# Legal Cold-Start

一个最小的法律 Agent 冷启动机制。在新的法律工作目录下打开 Claude Code，自动检测未配置状态，通过对话快速建立实践档案。

## 这是什么

把 Claude Code 变成你的法律助手之前，它需要知道：你是谁、公司做什么、在哪个法域、用什么工具、偏好什么输出风格。这三份文件做这件事：

- **CLAUDE.md** — 启动入口。每次对话自动加载，检测档案状态，触发冷启动或直接工作
- **legal-profile.md** — 实践档案。冷启动对话的结果，此后每次对话模型都读取它
- **cold-start-script.md** — 冷启动脚本。对话大纲、种子文件提取指南、档案模板。仅在未配置时加载

## 怎么用

1. 把这个目录作为你的法律工作区根目录，或者在现有工作区中放入这三份文件
2. 在此目录下打开 Claude Code
3. 模型自动检测到档案未配置，询问是否开始冷启动
4. 你说"好"，3-5 分钟对话完成配置
5. 此后每次打开，模型静默读取档案，按你的偏好工作

## 冷启动过程中会发生什么

模型会：

1. 问你要种子文件（合同模板、常用文书范本），如果有就从中提取业务领域、法域、风格等信息
2. 确认提取结果，纠正不准确的
3. 问文件看不出来的（法务部规模、审批链、工具偏好、输出风格）
4. 实际测试你声明的每个工具是否真的可用（不是读配置，是真调用）
5. 总结确认，写入档案

此后随时说 "update my profile" 可以逐项修改。

## 依赖

- Claude Code
- 无其他依赖。工具（yd-law、qcc、agentdocx 等）由你自己安装配置

## 设计思想

详见 `cold-start-script.md` 中的对话大纲和种子文件提取指南。核心思路来自 [claude-for-legal](https://github.com/anthropics/claude-for-legal) 的 cold-start-interview 设计。
