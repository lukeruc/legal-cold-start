# Legal Cold-Start

一个最小的法律 Agent 冷启动机制。在新的法律工作目录下打开 Claude Code，自动检测未配置状态，通过对话快速建立实践档案。

## 这是什么

把 Claude Code 变成你的法律助手之前，它需要知道你是谁、做什么、在哪个法域、用什么工具、偏好什么输出风格。这个项目做这件事：

- **CLAUDE.md** — 启动入口。每次对话自动加载，检测档案是否存在，触发冷启动或直接工作。同时是共享 guardrails
- **`.claude/legal-profile.md`** — 实践档案。冷启动对话的产物，此后每次对话模型都读取它。用户可直接编辑

## 怎么用

1. 把这个目录作为你的法律工作区根目录，或者在现有工作区中放入这些文件
2. 在此目录下打开 Claude Code
3. 模型检测到 `.claude/legal-profile.md` 不存在，问你的角色：公司律师、律所律师、还是学术研究者
4. 根据你的选择，模型读取对应的冷启动脚本，3-5 分钟对话完成配置
5. 此后每次打开，模型静默读取档案，按你的偏好工作

## 冷启动过程中会发生什么

1. 模型问你的角色（三选一）
2. 根据角色读取对应脚本
3. 问你要种子文件（合同模板、起诉状范本、论文草稿——不同角色不同类型），如果有就从中提取业务领域、法域、风格
4. 确认提取结果，纠正不准确的
5. 问文件看不出来的（团队分工、工具偏好、输出风格、公司政策/红线……）
6. 实际测试你声明的每个工具是否真的可用（不是读配置，是真调用）
7. 总结确认，写入 `.claude/legal-profile.md`
8. 清理安装文件（scripts/ 和 profiles/ 目录），留下一个干净的工作区

此后随时说 "update my profile" 可以逐项修改。

## 三种角色

| 角色 | 档案核心 | 冷启动会问 |
|------|---------|-----------|
| 公司律师 | 公司画像 + 内部政策 + 审批链 + 子公司 | 公司业务、法域、交易角色、公司红线、决策边界、关键内部人员 |
| 律所律师 | 执业画像 + 客户 + 方法论 + 利益冲突 | 业务领域、客户类型、办案风格、法域与法院、利益冲突管理 |
| 学术研究者 | 研究画像 + 方法论 + 发表策略 + 文献源 | 研究领域、方法论、目标期刊、引用格式、常用数据库 |

## 冷启动完成后

```
.
├── CLAUDE.md                    ← 启动逻辑 + guardrails（常驻）
├── README.md
├── LICENSE
├── .gitignore
└── .claude/
    └── legal-profile.md         ← 实践档案（唯一运行时文件）
```

scripts/ 和 profiles/ 是安装工具——配置完成后自动删除。
