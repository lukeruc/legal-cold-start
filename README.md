# Legal Cold-Start

让 Claude Code 在打开你的法律工作目录时，自动通过对话了解你是谁、做什么、怎么干活。

## 安装

你只需要 Claude Code。在终端中：

```bash
# 进入你的法律工作目录（或新建一个）
cd ~/my-legal-work

# 下载项目文件
git clone https://github.com/lukeruc/legal-cold-start.git .
```

或者手动操作：

1. 下载本项目的 ZIP
2. 把解压后的所有文件放到你的法律工作目录根目录下
3. 文件结构应该是：

```
你的法律工作目录/
├── CLAUDE.md            ← 启动入口
├── README.md
├── LICENSE
└── .claude/
    ├── scripts/          ← 冷启动脚本（配置后自动删除）
    │   ├── in-house.md
    │   ├── law-firm.md
    │   └── academic.md
    └── profiles/         ← 档案模板（配置后自动删除）
        ├── in-house.md
        ├── law-firm.md
        └── academic.md
```

**安装完成。** 在这个目录下打开 Claude Code 即可。

> ⚠️ **在开始之前**：你需要自己安装和配置法律数据库、文档处理等工具。本项目的冷启动会帮你验证它们是否真的可用，但不会替你安装。工具作为 Claude Code 的 skill 或 MCP 连接器安装，配置方式参见 Claude Code 文档。

## 首次使用

打开 Claude Code 后，模型检测到尚未配置，自动开始对话：

1. **问你的角色**：公司律师、律所律师、还是学术研究者
2. **根据角色读取对应脚本**，开始 3-5 分钟配置对话
3. **如果你有种子文件**（合同模板、起诉状范本、论文草稿），模型从中提取业务领域、法域、写作风格。没有就直接聊
4. **实际测试你的工具**：不是读配置文件，是真调用——确认 yd-law 能检索、qcc 能查询
5. **只问真正影响输出的**：公司政策红线、决策边界、客户类型、引用格式、输出风格。不问你团队多少人、不记录你手下名单
6. **总结确认，写入 `.claude/legal-profile.md`**，同时删除安装脚本——此后你的工作目录只有一个 `CLAUDE.md` 和一个干净 profile

此后每次打开，模型静默读取档案，按你的方式工作。说 "update my profile" 随时可修改。

## 三种角色

| 角色 | 档案记录什么 | 不记录什么 |
|------|-------------|-----------|
| 公司律师 | 公司业务、法域、交易角色、内部政策红线、决策边界、行业标准 | 子公司清单（抽象概括即可）、内部人员名单、当前事项 |
| 律所律师 | 执业领域、客户类型、法域与法院、诉讼/非诉文书风格、引用格式 | 办案风格标签、利益冲突规则、活跃案件列表 |
| 学术研究者 | 研究领域、研究法域、产出类型、目标期刊、引用格式、写作风格 | 方法论标签、文献源清单、当前项目 |

核心原则：档案只记会改变模型输出行为的信息。不记参考清单。

## 冷启动完成后

```
    └── legal-profile.md         ← 你的实践档案
```
