# Legal Cold-Start

## 这玩意儿解决什么问题

Claude Code 每次新对话是全新上下文——它不记得上一轮聊了什么，更不记得你是谁。你在每一次对话开头都要手动说明："我是公司法务，主要审工程合同，用中国大陆法律，输出偏实务，风险用🔴🟠🟡来标……"

**这个项目做的事：第一次花3-5分钟跟 Claude 聊完，它把你的身份、领域、法域、内部政策、输出偏好写到一个文件里。此后每次在这个目录打开 Claude Code，它自动读取这份档案——不用你再重复。**

不是让你"配一个 AI"，是让你**不用每次重复告诉 AI 你是谁**。

## 怎么用

```bash
cd ~/my-legal-work
git clone https://github.com/lukeruc/legal-cold-start.git .
```

在这个目录下打开 Claude Code。模型检测到还没配置，会问你的角色（公司律师/律所律师/学术研究者），然后开始3-5分钟对话。你可以给它种子文件（合同模板、起诉状范本、论文草稿），让它从文件里提取你的业务领域、法域和风格——也可以直接聊。

聊完以后，对话内容写入 `.claude/legal-profile.md`，安装脚本自动删除。此后每次打开这个目录，模型静默读取档案，按你的方式工作。

想改？说 "update my profile" 就行。或者直接编辑 `.claude/legal-profile.md`——它就是纯文本。

## 三种角色

冷启动第一个问题问你是哪种角色，后续问的问题完全不同：

| 角色 | 会了解 | 不会问 |
|------|--------|--------|
| 公司律师 | 公司业务、法域、交易角色、内部政策红线、审批链、行业标准 | 不问你手下几个人、不记子公司名单 |
| 律所律师 | 执业领域、客户类型、法域与常去法院、诉讼/非诉文书风格 | 不问你办案风格标签、不记案件列表 |
| 学术研究者 | 研究领域、研究法域、产出类型、目标期刊、引用格式 | 不问你用什么方法论、不记文献源清单 |

原则：**只记能改变模型输出行为的信息**。参考清单和通讯录不塞进档案。

## 工具配置

> ⚠️ 你需要自己安装法律数据库、工商查询、文档处理等工具。冷启动会帮你实际测试它们是否连得上（不是读配置文件，是真调用），但不会替你装。
>
> 文档处理推荐 [agentdocx](https://github.com/lukeruc/agentdocx)——专门为法律文档设计的 .docx 读写与修订 skill，支持 tracked changes 和批注。

## 冷启动完成后

```
你的法律工作目录/
├── CLAUDE.md                    ← 启动逻辑 + 共享规则
└── .claude/
    └── legal-profile.md         ← 你的档案（纯文本，可手改）
```

安装时的 scripts/ 和 profiles/ 目录已自动删除。留下的只有这两个文件。
